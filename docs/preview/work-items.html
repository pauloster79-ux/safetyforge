Work Item UI Design

From conversation to structured work · Scales from solo to 100-person crew

The Core Idea

A WorkItem is the atomic unit of deliverable work. It has a description, optional quantity, labour estimate, material estimate, and a lifecycle state. The user creates them by talking — the agent extracts structured items from natural language and presents them as editable cards for review.

The card IS the work item. It appears in chat as a draft. The user can confirm it (one tap), edit any field (tap the value), or dismiss it. On desktop, the same card appears in the canvas table view. There's no separate "create form" — the conversation IS the form.

Work Item Lifecycle

StateCard AppearanceWhat Happens
DraftYellow left border. Editable.Agent extracted from conversation. Awaiting user review.

In ProgressShows est. vs actual time.Work underway. Time entries logging against it.
CompleteMuted. Shows final vs estimated.Work done. Actual cost known.

Work Package

A WorkPackage is an optional grouping. It emerges naturally — when Sarah says "reception area" the agent creates a package. Or the user can group items later. Solo contractors like Jake never see packages — their items are flat on the project.
