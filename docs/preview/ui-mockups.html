Kerf UI Architecture

Conversational-first · Phone · Tablet · Desktop

Interaction Model

Chat is the home screen. Everything else is a view the chat produces. The user doesn't navigate to "Estimates" — they say "show me the estimate for 4th Street" and it appears as an interactive card. Hamburger menu provides traditional navigation as a secondary path.

Three Ways to Do Anything

MethodHowWhen
ChatType a messageEverywhere — phone, tablet, desktop
VoiceTap mic (STT) or real-time conversation (LiveKit)On site, driving, hands busy
NavigateHamburger menu → traditional screensAdmin, review, preference

Fixed vs Generated Views

Pre-built (fast, always available): Morning brief, project list, clock-in, daily log review, active inspections

Agent-generated (from conversation): Estimates, cost reports, schedule views, compliance checks, worker profiles, equipment status, ad-hoc queries

WhatsApp: Worker Comms

Kerf messages workers via WhatsApp · No app install required
The contractor tells Kerf to message a worker, or Kerf detects it automatically (cert expiring, schedule change). The message goes via WhatsApp Business API. Responses are captured in the knowledge graph. Cost: ~$0.004-0.02 per message.
